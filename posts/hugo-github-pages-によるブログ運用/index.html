deploy について

以下のような簡易 script を実行し、submodule 管理している public 配下の差分を commit & push することで deploy 完了。 blog のソース、設定ファイル等を private レポジトリで管理、public(blog 本体)配下のみをパブリックなレポジトリで管理できるようにしている。

#!/bin/bash
set -ex

# Build.
hugo

# Go To Public folder
pushd public

# push to public repo
git add .
git commit -m "build source at $(date)."
git push origin HEAD
popd
